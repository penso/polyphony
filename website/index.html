Get started in seconds


      
      

        

          
          
          
          
        


        
        

          

            

              

              

              

            

            

              
                $ brew install penso/polyphony/polyphony
              
              
            

          

          
Universal binary (Apple Silicon + Intel)

        


        
        

          

            

              

              

              

            

            

              
                $ curl -fsSL https://raw.githubusercontent.com/penso/polyphony/main/install.sh | sh
              
              
            

          

          
Auto-detects x86_64/aarch64. Downloads pre-built binary from GitHub Releases.

        


        
        

          

            

              

              

              

            

            

              
                > irm https://raw.githubusercontent.com/penso/polyphony/main/install.ps1 | iex
              
              
            

          

          
Pre-built binary for x86_64. Or download the .zip from GitHub Releases.

        


        
        

          

            

              

              

              

            

            

              

                
                  $ git clone https://github.com/penso/polyphony.git
                
              

              

                
                  $ cd polyphony && cargo +nightly build --release -p polyphony-cli
                
              

            

          

          
Requires Rust nightly. Works on any platform Rust supports.

        

      

    

  


  
  

    

      

        
        

          
🎯

          
Issue → Agent → PR

          

            Connect trackers, dispatch agents, get pull requests automatically. From issue to shipped code, hands-free.
          

        

        
        

          
🤖

          
Multi-Agent

          

            Claude, Codex, Copilot, Pi — use whichever agent fits the task. Mix and match with fallback chains.
          

        

        
        

          
📦

          
Repo-Native

          

            Inspired by OpenAI Symphony, workflow policy lives in WORKFLOW.md — version-controlled with your code. No external config servers.
          

        

      

    

  


  
  

    

      
How it works

      

        Four steps from issue to shipped code.
      


      

        
        

          
1

          

          
Configure

          
Point to your tracker (GitHub, GitLab, Linear) and choose agents.

          

            

              
tracker: github

              
agents:

              
  - claude

              
  - codex

              
workspace: worktree

            

          

        


        
        

          
2

          

          
Orchestrate

          
Polyphony polls for issues, creates isolated workspaces, and dispatches agents.

          

            
⠋ Polling tracker...

            
✓ Found 3 new issues

            
→ Creating worktree for #42

            
→ Dispatching claude to #42

          

        


        
        

          
3

          

          
Monitor

          
Live TUI dashboard shows agent progress, retries, and results in real time.

          

            
+- Movements -----------------------------+
| ~ #42 Fix auth refresh  claude  running |
| * #43 Add rate limits   codex   done    |
| . #44 Update docs       pi      queued  |
+------------------------ 1 of 3 ---------+

          

        


        
        

          
4

          
Ship

          
Agents commit, push, and open PRs. Review and merge.

          

            
✓ Agent committed 3 files

            
✓ Pushed to fix/auth-refresh

            
→ PR #87 opened: "Fix auth token refresh"

            
  Waiting for review...

Built for real workflows

      

        Everything you need to run AI agents in production.
      


      

        

          
⎔

          
Isolated Workspaces

          
Each agent gets its own git worktree. No conflicts, no interference, clean diffs.

        

        

          
↻

          
Retry & Fallback

          
Automatic retry with configurable agent fallback chains. If Claude fails, try Codex.

        

        

          
⊘

          
Budget Control

          
Token and cost-aware throttling. Set concurrency limits and per-issue budgets.

        

        

          
⬡

          
Workflow Policy

          
YAML + Liquid templates in WORKFLOW.md. Define routing, priority, and agent selection rules.

        

        

          
◉

          
Live Observability

          
Real-time TUI with snapshots and non-blocking updates. Never miss an agent's progress.

        

        

          
⇒

          
Handoff Automation

          
Auto-commit, push, PR creation, and feedback loops. Agents ship code, not just write it.
